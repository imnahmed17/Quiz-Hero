Important Questions!!!

  • Question 1: What are the differences between map, filter, forEach and find?
  • map() is used to create a new array from an existing array. It iterates through each element in the array and applies the provided function to each element and returns a new array. It doesn't alter the original array.
  • filter() is used to create a new array from an existing array by filtering out unwanted elements. It iterates through each element of the array and returns a new array with only the elements that pass the filter criteria.
  • forEach() is used to iterate through an array and perform an action on each element. It does not return a new array, but instead applies a function to each element of the existing array.
  • find() is used to find a single element in an array that matches a given criteria. It iterates through each element of the array and returns the first element that matches the given criteria.
  • Question 2: What are the differences between local storage, session storage and cookies?
  • Local storage allows web applications to store data locally within the user's browser. It is permanent and remains even when the browser is closed. It can store up to 10MB of data.
  • Session storage is similar to local storage, but it is limited to the current session. It is deleted when the user closes the browser. It can store up to 5MB of data.
  • Cookies are small pieces of data stored in the user's browser. They are usually used to store information like login credentials, preferences, and shopping cart items. They are usually limited to 4KB of data and can be deleted by the user at any time.
  • Question 3: What is event loop and how does it work?
  • Event Loop is the secret behind JavaScript's asynchronous programming. It has one simple job to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop. Each event is just a function callback. It works by repeatedly checking for new events or inputs, and then processing them in order. The Event Loop is responsible for responding to user input, refreshing the display, and handling communication with other systems.
  • Question 4: What is use of "quizTimer()" function in this project?
  • The quizTimer() function is used to keep track of the time a user takes to answer the quizzes. The function also takes a dismiss argument, if it is set to true, will stop the counting. Initially, the color of the timer is green but when the value of the timer exceeds 60 seconds the color of the timer turns red.
  • Question 5: What is use of "chooseQuiz()" function in this project?
  • The chooseQuiz() function takes an index and a given answer as arguments and updates the answers array for the selected quiz from the quizData array. It checks whether the quizzes answer already exists in the answers array and if it does, it replaces the existing answer with the given answer. If not, it adds the quizzes answer to the answers array. It also calls the displayAnswers function to display the updated answers array.
